# 마우스 궤적 생성 모델 평가 계획

이 문서는 생성된 마우스 궤적이 실제 사람 데이터와 얼마나 비슷한지, 그리고 실제로 사람다운 궤적인지 평가하는 방법을 정리한 계획입니다.

평가는 크게 두 축으로 나눕니다.

```text
1. 실제 데이터와 통계적으로 얼마나 비슷한가
2. 사람이 보기에도 사람다운 궤적인가
```

즉 숫자로 보는 평가와 시각적/지각적 평가를 함께 사용해야 합니다.

## 1. 기본 궤적 형태 비교

먼저 생성 궤적과 실제 궤적의 모양이 비슷한지 비교합니다.

비교할 수 있는 값은 다음과 같습니다.

```text
최대 이탈량
직선 경로에서 가장 멀리 벗어난 정도

평균 이탈량
전체 경로가 평균적으로 얼마나 휘었는지

곡률
경로가 얼마나 부드럽게 휘는지

총 이동거리
시작점에서 목표점까지의 직선거리보다 얼마나 더 많이 움직였는지

overshoot
목표점을 지나쳤다가 돌아오는 패턴이 있는지
```

사람 마우스는 완전한 직선이 아닙니다. 하지만 너무 많이 흔들리거나, 너무 기계적으로 매끈해도 부자연스럽습니다.

따라서 실제 데이터와 생성 데이터에서 위 값들의 분포를 비교합니다.

예를 들어 다음처럼 볼 수 있습니다.

```text
실제 데이터의 평균 최대 이탈량: 0.08
생성 데이터의 평균 최대 이탈량: 0.075
```

두 값이 비슷하면 생성 궤적의 형태가 실제 데이터와 잘 맞는 것입니다.

반대로 생성 데이터의 최대 이탈량이 거의 항상 `0.0`이면 모델이 너무 직선만 만든다는 뜻입니다.

## 2. 속도 프로파일 비교

사람 마우스 움직임은 보통 특정한 속도 패턴을 가집니다.

대체로 다음과 같은 흐름이 많습니다.

```text
처음에는 천천히 출발
중간에서 빨라짐
목표 근처에서 감속
```

즉 속도 곡선이 종 모양에 가까운 경우가 많습니다.

비교할 수 있는 값은 다음과 같습니다.

```text
peak velocity
최고 속도

time to peak velocity
최고 속도에 도달하는 시점

average velocity
평균 속도

acceleration profile
가속도 패턴

jerk
가속도의 변화량, 움직임이 얼마나 덜컥거리는지
```

사람다운 궤적은 위치만 자연스러운 것이 아니라 시간에 따른 속도 변화도 자연스러워야 합니다.

생성 모델이 모양은 괜찮게 만들었더라도 속도가 거의 일정하면 사람답지 않습니다.

```text
기계적 이동
속도가 거의 일정함

사람 이동
중간에서 빠르고 목표 근처에서 느려짐
```

따라서 실제 데이터와 생성 데이터의 속도 프로파일을 비교해야 합니다.

## 3. 조건별 비교

전체 평균만 비교하면 부족합니다.

현재 모델은 조건부 생성 모델이므로, 조건별로 잘 맞는지 봐야 합니다.

모델이 사용하는 조건은 다음과 같습니다.

```text
A         이동 거리
W         목표 크기
ID        난이도
duration  이동 시간
```

예를 들어 `ID`가 낮을 때와 높을 때 궤적이 달라져야 합니다.

일반적으로 다음과 같은 경향을 기대할 수 있습니다.

```text
ID 낮음
짧고 쉬운 이동, 비교적 안정적

ID 높음
길거나 목표가 작음, 더 조심스럽고 흔들림이나 감속 증가 가능
```

따라서 조건별 평가는 다음처럼 나눠서 진행할 수 있습니다.

```text
ID 구간별 최대 이탈량 비교
A 구간별 총 이동거리 비교
W 구간별 목표 근처 흔들림 비교
duration 구간별 속도 프로파일 비교
```

이렇게 해야 모델이 전체적으로 그럴듯한 평균 궤적만 만드는지, 아니면 조건에 맞게 궤적을 변화시키는지 확인할 수 있습니다.

## 4. 분포 비교

생성 모델은 하나의 평균 궤적만 맞추면 부족합니다.

중요한 것은 실제 사람 데이터의 다양성과 생성 궤적의 다양성이 비슷한지입니다.

비교 방법은 다음과 같습니다.

```text
특징량별 히스토그램 비교
실제 vs 생성 KDE 비교
KS test
Wasserstein distance
MMD
```

비교할 특징량 예시는 다음과 같습니다.

```text
총 이동거리
최대 이탈량
평균 이탈량
최고 속도
최고 속도 시점
평균 곡률
jerk
overshoot 횟수
목표 근처 미세 흔들림
```

예를 들어 실제 데이터의 최고 속도 분포가 넓은데, 생성 데이터의 최고 속도 분포가 매우 좁으면 모델이 다양성을 충분히 만들지 못한 것입니다.

CVAE를 사용한 이유가 다양성 있는 생성이므로, 분포 비교는 특히 중요합니다.

## 5. 평균 궤적만 생성하는지 확인

생성 모델에서 흔한 실패는 평균 궤적만 생성하는 것입니다.

이 경우 생성 결과는 깨끗하고 그럴듯해 보일 수 있지만, 실제 사람 데이터의 다양성은 사라집니다.

확인 방법은 다음과 같습니다.

```text
같은 조건에서 여러 개 생성
생성 궤적들 사이의 분산 측정
실제 같은 조건 trial들의 분산과 비교
```

실제 사람 궤적의 분산보다 생성 궤적의 분산이 너무 작으면 다음과 같이 해석할 수 있습니다.

```text
모델이 너무 안정적인 평균 궤적만 만든다
```

반대로 생성 궤적의 분산이 실제보다 너무 크면 다음과 같이 볼 수 있습니다.

```text
조건과 상관없는 랜덤한 흔들림이 많다
```

## 6. 목표점 근처 행동 평가

사람다운 마우스 움직임에서 중요한 부분은 목표점 근처입니다.

사람은 목표에 가까워질수록 속도를 줄이고, 때로는 미세 조정을 합니다.

볼 수 있는 값은 다음과 같습니다.

```text
마지막 10~20% 구간의 속도
목표 근처 흔들림
목표를 지나친 정도
목표에 들어오기까지의 경로
최종 접근 각도
```

특히 `W`가 작을수록 목표 근처에서 더 조심스러운 패턴이 나오는지 보는 것이 좋습니다.

만약 목표 크기가 작은데도 생성 궤적이 큰 목표와 똑같이 빠르게 들어간다면, 조건을 제대로 반영하지 못한 것입니다.

## 7. Fitts' law 관점 평가

이 데이터가 Fitts task 기반이라면 Fitts' law 관점의 평가도 중요합니다.

Fitts' law는 대략 다음 관계를 봅니다.

```text
이동 시간 MT = a + b * ID
```

평가 방법은 다음과 같습니다.

```text
실제 데이터에서 ID와 duration 관계를 회귀
생성 데이터에서 ID와 duration 또는 생성 속도 관계를 회귀
두 회귀선 비교
```

다만 현재 생성 명령에서는 `duration`을 입력값으로 줍니다. 즉 현재 모델은 duration을 직접 예측하는 구조는 아닙니다.

그래도 다음은 평가할 수 있습니다.

```text
같은 duration 조건에서 ID가 높을 때 궤적 형태가 달라지는가
실제 데이터의 ID별 궤적 특성과 생성 데이터의 ID별 궤적 특성이 비슷한가
```

추후에는 모델이 `duration`도 직접 생성하거나 예측하도록 확장할 수 있습니다.

## 8. 시각화 평가

숫자만 보면 궤적의 자연스러움을 파악하기 어렵습니다.

따라서 시각화는 반드시 필요합니다.

좋은 시각화 예시는 다음과 같습니다.

```text
실제 궤적 여러 개 overlay
생성 궤적 여러 개 overlay
조건별 비교 plot
속도 프로파일 평균 ± 표준편차
이탈량 분포 histogram
곡률/jerk 분포
```

특히 같은 조건에서 실제 궤적과 생성 궤적을 겹쳐보는 것이 좋습니다.

예시는 다음과 같습니다.

```text
A=500, W=40, ID=3.75 조건의 실제 50개 궤적
A=500, W=40, ID=3.75 조건의 생성 50개 궤적
```

이렇게 보면 모델이 직관적으로 괜찮은지 빠르게 확인할 수 있습니다.

## 9. 판별기 기반 평가

조금 더 딥러닝스럽게 평가하려면 별도의 classifier를 만들 수 있습니다.

방법은 다음과 같습니다.

```text
실제 궤적에는 label=real
생성 궤적에는 label=fake
분류 모델을 학습
```

만약 classifier가 real/fake를 쉽게 구분한다면 생성 궤적이 아직 실제와 다르다는 뜻입니다.

반대로 classifier 정확도가 50% 근처라면 다음과 같이 해석할 수 있습니다.

```text
기계가 보기에도 실제와 생성이 잘 구분되지 않는다
```

다만 이 방식은 classifier의 성능과 편향에 영향을 받습니다. 따라서 단독 지표로 쓰기보다는 보조 지표로 쓰는 것이 좋습니다.

## 10. 사람 대상 평가

최종적으로 사람다운지는 사람에게 물어보는 것도 좋습니다.

실제 궤적과 생성 궤적을 섞어 보여주고 다음을 평가하게 할 수 있습니다.

```text
어느 것이 실제 사람 궤적인가?
자연스러움 점수는 몇 점인가?
기계적으로 보이는가?
```

가능한 방식은 다음과 같습니다.

```text
A/B test
real/fake 구분 테스트
Likert scale 자연스러움 평가
```

만약 사람이 생성 궤적을 실제와 잘 구분하지 못하면, 생성 모델이 사람다운 궤적을 만들고 있다는 강한 근거가 됩니다.

## 11. 평가 세트 구성

학습에 쓴 데이터와 평가 데이터를 반드시 분리해야 합니다.

추천 구조는 다음과 같습니다.

```text
train set
모델 학습용

validation set
학습 중 튜닝용

test set
최종 평가용
```

특히 사람 단위로 나누는 평가도 의미가 있습니다.

```text
seen subject 평가
학습에 등장한 사람의 새로운 trial 평가

unseen subject 평가
학습에 등장하지 않은 사람의 trial 평가
```

`unseen subject`에서도 괜찮은 결과가 나오면 모델이 특정 사람에게만 과적합되지 않고 일반화된 패턴을 배웠다고 볼 수 있습니다.

## 12. 추천 평가 지표 조합

처음에는 너무 복잡하게 시작하기보다 다음 조합을 추천합니다.

```text
형태 지표
총 이동거리, 최대 이탈량, 평균 이탈량, 곡률

속도 지표
최고 속도, 최고 속도 시점, jerk

조건별 지표
ID 구간별, A/W 구간별 비교

분포 지표
KS test 또는 Wasserstein distance

시각화
실제 vs 생성 overlay, 속도 프로파일 비교

사람 평가
real/fake 구분 또는 자연스러움 점수
```

## 최종 판단 기준

이 모델의 평가는 단순히 MSE가 낮은지로 판단하면 부족합니다.

마우스 궤적 생성에서 중요한 것은 다음입니다.

```text
실제 궤적의 평균 형태와 비슷한가
속도 변화가 사람다운가
조건 변화에 맞게 궤적도 변하는가
다양성이 실제 사람 데이터와 비슷한가
목표점 근처 행동이 자연스러운가
사람이나 classifier가 real/fake를 구분하기 어려운가
```

따라서 최종 평가 프레임은 다음과 같이 잡는 것이 좋습니다.

```text
통계적 유사성
+ 시간적 자연스러움
+ 조건 반응성
+ 다양성
+ 시각/사람 평가
```

이렇게 평가하면 모델이 단순히 그럴듯한 선을 만든 것인지, 실제로 사람다운 마우스 궤적을 생성하는지 더 설득력 있게 판단할 수 있습니다.
